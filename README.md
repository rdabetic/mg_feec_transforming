# Multigrid for FEEC using Mass-Lumping and Transforming Smoothers

Clone with `--recursive` to get all the submodules.

The main driver is `targets/cvg.cpp`, which is built as the `cvg` executable.

## Build requirements

- CMake 3.28 or newer
- A C++ compiler supporting C++23 and OpenMP
- Boost with `program_options`
- System Eigen if available; otherwise the bundled `extern/eigen` fallback is used
- SuiteSparse headers installed at `/usr/include/suitesparse` or available to MFEM
- Git submodules:
  - `extern/eigen`
  - `extern/spectra`
  - `extern/mfem`
  - `extern/googletest` (for tests)
- Optional: SLURM to run `runme.py` on a cluster

## Build instructions

From the repository root:

```bash
cmake -B release -S . -DCMAKE_BUILD_TYPE=Release
cmake --build release -j 16
```

This will build the project (with tests) and the `cvg` executable at:

```bash
release/targets/cvg
```

## Running the convergence tests (`cvg`)

The `cvg` executable reads a JSON configuration file and executes convergence experiments. Example:

```bash
./release/targets/cvg --config configs/<name>.json
```

The default configuration directory is `configs/`, which contains a bunch of JSON files describing parameters for individual runs.
See also the help message of `cvg`.

## Results and Logs

`runme.py` is a cluster-oriented wrapper for launching `cvg` experiments using SLURM. It reads JSON configs from `configs/`, builds the project if needed, and submits the corresponding jobs. Only intended for use with SLURM on an HPC cluster.

If you only want to run a single convergence experiment, use `./release/targets/cvg --config ...` directly, no SLURM needed.

Read `gen_configs.py` if you want to modify the automatic generation of config files (it is NOT run automatically by `runme.py`).

The results from the experiments generated by `runme.py` are:

- `out/` contains CSV result files generated by `cvg`
- `cluster_logs/` contains SLURM submission logs and per-job stdout/stderr for `runme.py`

They can then be plotted using `plot_mg.py`.

## Minimal reproducible example

A very small convergence experiment can be run directly with a simple JSON config and the built `cvg` executable. For example:

```bash
cat > minimal_cvg.json <<'EOF'
{
  "problem": "mag",
  "mesh_file": "meshes/2d/L.msh",
  "output_file": "dummy_experiment.csv",
  "max_ref": 2,
  "num_eigs": 1,
  "runs": 1,
  "lumping": "rowsum",
  "cycle": "v",
  "eig_tol": 0.01,
  "bcond": "essential",
  "krylov_dim": 20,
  "gmres_tol": 1e-06,
  "gmres_maxit": 128,
  "ignore_gmres_fail": false,
  "mg_iters": 16,
  "verbose": true
}
EOF

./release/targets/cvg --config minimal_cvg.json
```
